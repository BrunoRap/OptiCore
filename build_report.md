## 5. Build Errors Encountered and Resolutions

### Error 1 — `CS1069: ServiceController not found`
**File:** `src/Services/MetricsService.cs`  
**Cause:** Used `System.ServiceProcess.ServiceController` which requires an explicit assembly reference not available without adding `System.ServiceProcess` NuGet package.  
**Resolution:** Replaced with a `Process.Start("powershell.exe", "...")` call to query service count inline, eliminating the dependency.

### Error 2 — `CS1069: ServiceController not found` (second instance)
**File:** `src/Views/ValidateView.xaml.cs`  
**Cause:** Same `System.ServiceProcess.ServiceController` usage in the validation check helper.  
**Resolution:** Replaced with a direct `Registry.LocalMachine.OpenSubKey("SYSTEM\CurrentControlSet\Services\{name}")` read — checks the `Start` value (4 = Disabled/Stopped) without needing the ServiceController class.

### Warning batch — `CS0618: XUnit implicit conversion obsolete` (PdfSharp 6.2)
**File:** `src/Services/ReportService.cs`  
**Cause:** PdfSharp 6.1+ deprecated implicit `double ↔ XUnit` conversions (changed to require explicit `.Point` property access).  
**Resolution:** Changed `page.Width` / `page.Height` usages to `page.Width.Point` / `page.Height.Point`. All 8 warnings resolved.

## 9. Known Issues and Limitations

| # | Issue | Severity | Impact |
|---|---|---|---|
| 1 | **No BTT.ico** — App icon shows as generic WPF icon in taskbar and EXE properties | Low | Cosmetic only |
| 2 | **Language not persisted** — Selected language resets to English on next launch | Medium | UX regression; AppSettings model exists, just needs JSON serialize/deserialize wiring |
| 3 | **Razer HID command is a stub** — `razer_polling` PS case logs "success" without actually sending the HID feature report | Medium | Razer keyboard IRQ fix won't work; needs raw USB HID bytes from original script |
| 4 | **ValidateView covers 8 of 29 optimizations** — 21 checks not yet implemented | Medium | Validation is incomplete; user can't verify all applied settings |
| 5 | **PBO OC mode detection is heuristic** — Checks one registry key; boards that implement PBO differently may report as Stock | Low | Incorrect PBO warning suppressed on procthrottle_min; safe fallback behavior |
| 6 | **GPU interrupt affinity may need manual PCI path** — The NVIDIA device enumeration iterates all VEN_10DE subkeys; on multi-GPU systems it will set affinity on all NVIDIA devices | Low | Harmless on single-GPU systems (RTX 5090 only) |
| 7 | **XHCI suspend uses Get-PnpDevice name match** — FriendlyName filter for "xHCI" may miss USB 3.2 Gen 2 controllers with different naming | Low | Some USB controllers may not be configured; reboot check won't find them |
| 8 | **BitLocker / fTPM detection uses manage-bde** — Requires manage-bde.exe to be in PATH; on some systems it may not run without elevation | Low | BitLocker/TPM fields show blank rather than "Active/Inactive" |
| 9 | **Metrics fallback runs without PS script** — If publish directory layout changes, inline PerformanceCounter fallback runs. IRQ/DPC values will be different format | Low | Metrics comparison still works; timer resolution shows 15.625ms |
| 10 | **NIC EEE uses `*EEE` registry property name** — Some NIC drivers use different property names (e.g., `EEELinkAdvertisement`, `EnergyEfficientEthernet`) | Low | EEE may not be disabled on all NIC brands; Intel NICs known to work |
